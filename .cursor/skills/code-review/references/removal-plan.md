# 可删除项与迭代计划模板

## 优先级

- [ ] **P0**：需立即删除（安全风险、显著成本、阻塞其他工作）
- [ ] **P1**：当前迭代内删除
- [ ] **P2**：待办 / 下一迭代

---

## 可立即删除

### 项：[名称/描述]

| 字段 | 说明 |
|------|------|
| **位置** | `path/to/file.ts:line` |
| **理由** | 为何应删除 |
| **依据** | 未使用（无引用）、死 feature flag、已弃用 API |
| **影响** | 无 / 低 - 无活跃消费者 |
| **删除步骤** | 1. 删除代码 2. 删除测试 3. 删除配置 |
| **验证** | 跑测试、确认无运行时错误、监控日志 |

---

## 延后删除（需计划）

### 项：[名称/描述]

| 字段 | 说明 |
|------|------|
| **位置** | `path/to/file.ts:line` |
| **为何延后** | 有活跃消费者、需迁移、需干系人确认 |
| **前置条件** | Feature flag 关闭 2 周、遥测显示 0 使用 |
| **破坏性变更** | 列出 API/契约变更 |
| **迁移计划** | 消费者迁移步骤 |
| **时间线** | 目标日期或迭代 |
| **负责人** | 人或团队 |
| **验证** | 确认安全删除的指标（错误率、使用量） |
| **回滚** | 若发现问题如何恢复 |

---

## 删除前检查

- [ ] 在代码库中搜索所有引用（`rg`、`grep`）
- [ ] 检查动态/反射引用
- [ ] 确认无外部消费者（API、SDK、文档）
- [ ] 已审查 Feature flag 遥测（若适用）
- [ ] 测试已更新/删除
- [ ] 文档已更新
- [ ] 已通知团队（若为共享代码）
