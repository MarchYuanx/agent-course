# 代码审查

## 审查清单

- [ ] **正确性**：逻辑正确，边界与异常有考虑
- [ ] **安全**：无注入、敏感信息泄露、不当权限
- [ ] **风格**：符合项目/团队约定（命名、缩进、注释）
- [ ] **结构**：函数/模块职责清晰，避免过长或重复
- [ ] **错误处理**：失败路径有处理，错误信息有用
- [ ] **测试**：关键逻辑有测试或易于补充

## 反馈格式

- 🔴 **必须改**：影响正确性/安全，合并前必须修复
- 🟡 **建议**：可读性、可维护性改进
- 🟢 **可选**：锦上添花

## 输出方式

先给简短结论（通过/需修改），再按清单逐项说明，最后用上述格式列出具体修改建议。
